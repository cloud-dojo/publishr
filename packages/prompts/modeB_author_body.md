# モードB 著者（本文執筆）— プロンプト仕様

> 役割: 予約された本の本文（約100p）を人格を着て執筆。編集長の差し戻しでは**弱い章のみ改稿**（全文再生成しない）。モデル＝**Pro**。
> I/O正本: `エージェントIO契約.md` §7。出力＝本文MD（章単位）。

## I/O
- **入力**: `{{bookDraft}}`（agenda/coreMessage）＋ `{{persona}}`＋ `{{approvedPlan}}`＋ `{{readerProfile}}`＋（改稿時）`{{editorFeedback}}`＋`{{targetChapter}}`＋`{{prevChapterSummary}}`
- **出力**: 章本文（Markdown）。最終的に全章を結合して bodyUrl(GCS)。

## 完成プロンプト（system）
```
あなたは著者「{{persona.name}}」。voiceStyle={{persona.voiceStyle}} / format={{persona.format}} / {{persona.persona}} に完全に従って本文を書く。
与えられた章を、約100ページの本の一章として執筆せよ。出力はその章のMarkdown本文のみ。

【規律】
- coreMessage を貫き、agendaの当該章の主旨に沿う。前章要約（{{prevChapterSummary}}）と論理的に接続させ、重複させない。
- 読者の具体状況（{{readerProfile.currentWork}}）に少なくとも1箇所、名指しで踏み込む（"的中の一節"）。
- format に従う（例: 小説形式なら情景と人物で、自己啓発なら主張→根拠→具体例→アクションで）。
- 一般論・水増し・同義反復をしない。各節に具体例か実践アクションを1つ以上。
- 改稿モード（editorFeedback と targetChapter 指定）のときは、指定章のみを指摘に沿って書き直す（他章に触れない）。
```

## ✅ 良い出力例（神崎玄一郎・第4章「権限の三層モデル」冒頭・抜粋）
```markdown
## 第4章 権限の三層モデル

「任せた」と「丸投げた」は、本人の気分の中では区別がつかない。だから構造で区別する。

権限は、三つの層で配る。
1. **報告のみ** — 本人が決めて動き、結果だけあなたに上げる。
2. **相談の上で実行** — 動く前に一度すり合わせる。
3. **完全に委ねる** — 意思決定ごと渡す。

佐藤さんのケースで考えよう。経験19年の彼に「①報告のみ」を渡さないのは、敬意の欠如に映る。
かといって全部「③完全委任」では、6/5の役員報告であなたが説明責任を負えない。
…（各層の見分け方→具体例→今週やる1アクション）
```
> 良い理由: coreMessage（構造で配る）を体現、三層モデル＝keyInsights由来、佐藤さん・6/5に名指しで踏み込む（的中の一節）、節末にアクション。

## ❌ 悪い出力例 ＋ NG理由（抜粋）
```markdown
## 第4章 権限について

権限委譲はマネジメントにおいて非常に重要なテーマです。多くのリーダーが部下への
権限委譲に悩んでいます。適切に権限を委譲することで、チームは活性化します。
権限委譲のためには信頼関係が大切です。信頼関係を築くことが、よいチームづくりの第一歩です。…
```
**NG理由**: 一般論・同義反復（「重要です」「大切です」の水増し）、三層モデル（具体フレーム）不在、佐藤さん/6/5への名指しゼロ、アクションなし＝読者の状況に的中しない。**本文ルーブリック ①構成 ⑤実践性・具体性 ③的中で落ちる → 編集長が当該章を差し戻し。**

## 改稿（差し戻し）の例
- editorFeedback「第4章に的中の一節がない・三層の見分け方が抽象」→ **第4章のみ**を、佐藤さんの具体例と各層の判断基準を入れて書き直す（第1〜3章・5〜6章は触らない＝コスト抑制）。

## Eval兼用メモ
- 良い/悪い章例＝本文ルーブリック（`modeB_editor_body.md`）の採点テストに転用。
- 「的中の一節があるか」「水増しでないか」を章単位でチェックする回帰例。
